# TBU Web Programming Basic (HTML/CSS)

This repository contains practical materials for the **Web Programming Basic** course focused on **HTML and CSS**.

## Course Goal

By the end of this course, students should be able to:

- Understand basic web page structure with HTML
- Style and layout pages with CSS
- Build responsive pages for desktop and mobile
- Organize small frontend projects clearly

## Course Structure (4 Lessons)

### Lesson 1: HTML Fundamentals

- HTML document structure (`<!DOCTYPE html>`, `<html>`, `<head>`, `<body>`)
- Common tags: headings, paragraphs, lists, links, images
- Semantic elements: `header`, `main`, `section`, `article`, `footer`

### Lesson 2: CSS Fundamentals

- Selectors, colors, typography, spacing
- Box model: `margin`, `border`, `padding`, `content`
- Basic styling best practices and reusable classes

### Lesson 3: Bootstrap 5 Basic

- Bootstrap 5 setup (CDN and project structure)
- Bootstrap grid system and responsive breakpoints
- Common components: navbar, buttons, cards, forms

### Lesson 4: Bootstrap 5

- Advanced layout composition with Bootstrap utilities
- Responsive navigation and page sections
- Build and present a complete Bootstrap-based webpage

## Students Workspace (`students/`)

Each student can create their own folder inside `students/` for practice and experiments, for example `students/lesson_01/`.

Important:

- The `students/` folder is ignored by Git (configured in `.gitignore`).
- This means students can freely add, edit, and delete files there without creating repository changes.
- Course lesson content in `Lesson_01` to `Lesson_04` should remain unchanged unless instructed.

## Tools

- Any modern browser (Chrome, Edge, Firefox)
- VS Code (recommended)
- PyCharm (recommended for beginners, especially for the Django part of the course)
- Optional: Live Server extension for faster preview

## How to Use This Repository

1. Open the repository in your code editor.
2. Enter the lesson folder.
3. Complete the tasks and examples for that lesson.
4. Test your code in the browser.
5. Use `students/lesson_01/` for personal practice files.

## Academic Integrity

- You may discuss ideas with classmates.
- Submitted code must be your own work.
- Copy-paste from the internet without understanding is not accepted.
